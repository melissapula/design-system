# mfp-design-system

A Lit-based design system, organized as a pnpm monorepo. Every package publishes to npm under the **`@mfp-design-system`** scope and works in any framework that supports custom elements (Vue, Nuxt, Angular, React 19+, plain HTML).

Each component:

- Renders into a Shadow DOM so app styles can't leak in or out by accident
- Consumes design tokens via CSS custom properties (with safe fallback values)
- Ships with TypeScript types, Storybook stories, and a Web Test Runner suite
- Is themeable ‚Äî see [Themes](#themes) below

## What's in here

### Tokens

**[`@mfp-design-system/tokens`](?path=/docs/themes-showcase--docs)** ‚Äî primitives the components are built on. Colors (base ramps, semantic, status), spacing, radius, typography, shadows, motion, z-index, breakpoints, letter-spacing, and a semantic brand layer that drives theming.

### Components

23 component packages, grouped by role:

**Action**

- `mfp-button` ‚Äî primary / secondary / danger / ghost
- `mfp-icon-button` ‚Äî icon-only, ghost-by-default, requires `aria-label`

**Form input**

- `mfp-input` ‚Äî single-line text (text / email / password / number / search / tel / url)
- `mfp-textarea` ‚Äî multi-line text
- `mfp-select` ‚Äî wraps native `<select>` for free a11y
- `mfp-checkbox` ‚Äî with `indeterminate` state
- `mfp-switch` ‚Äî toggle, `role="switch"`
- `mfp-radio` ‚Äî single radio; siblings sharing `name` auto-deselect

**Form wrapper**

- `mfp-form-field` ‚Äî label / hint / error wrapper for arbitrary controls

**Overlays**

- `mfp-modal` ‚Äî built on native `<dialog>`; focus trap + scroll lock for free
- `mfp-tooltip` ‚Äî hover / focus tooltip with 4 placements
- `mfp-toast` ‚Äî programmatic `showToast({ message, variant })` API

**Feedback**

- `mfp-alert` ‚Äî banner using status colors; optional dismiss
- `mfp-spinner` ‚Äî standalone loading indicator
- `mfp-badge` ‚Äî pill / chip in 6 variants

**Layout**

- `mfp-card` ‚Äî container with header / body / footer slots
- `mfp-divider` ‚Äî horizontal or vertical separator with optional label
- `mfp-avatar` ‚Äî image with initials fallback; optional status dot

**Organization**

- `mfp-tabs` ‚Äî `<mfp-tabs>` + `<mfp-tab>` + `<mfp-tab-panel>`; full keyboard nav
- `mfp-accordion` ‚Äî `<mfp-accordion>` + `<mfp-accordion-item>`; FAQ-style exclusive mode
- `mfp-stepper` ‚Äî `<mfp-stepper>` + `<mfp-step>`; status colors for completed / current / error

**Navigation & chrome**

- `mfp-nav` ‚Äî `<mfp-nav-bar>` + `<mfp-side-nav>` + `<mfp-nav-item>`; brand variant
- `mfp-footer` ‚Äî app footer shell; `default` / `brand` / `dark` variants

Browse them in the sidebar under **Components/**. Each story page has live controls, code examples, and an auto-generated docs page.

## Quick start (in any app)

```sh
npm install @mfp-design-system/tokens @mfp-design-system/button
```

```ts
// At your app's entry point
import '@mfp-design-system/tokens/css';
import '@mfp-design-system/tokens/themes/terracotta'; // pick a theme ‚Äî see below
import '@mfp-design-system/button';
```

```html
<mfp-button variant="primary">Save</mfp-button>
```

### Framework notes

- **Vue 3 / Nuxt** ‚Äî tell the compiler about the prefix:
    ```ts
    compilerOptions: {
        isCustomElement: (tag) => tag.startsWith('mfp-');
    }
    ```
- **Angular** ‚Äî add `CUSTOM_ELEMENTS_SCHEMA` to any module/standalone component that uses `<mfp-*>` tags
- **React 19+** ‚Äî works natively
- **Plain HTML** ‚Äî just use the tags

## Themes

Each consuming app can pick a brand theme that overrides the design system's default colors. Components consume semantic tokens (`--color-brand-primary`, `--color-brand-primary-hover`, etc.) so a single import re-themes every primary button, focus ring, checked state, and toggle.

```ts
import '@mfp-design-system/tokens/themes/terracotta'; // fourseasonsstudio brand
```

Theme names are the **color**, not a project ‚Äî `themes/navy` isn't locked to the portfolio, any app can pick any theme. Six themes ship today:

- **`blue`** ‚Äî default (matches "no theme"); explicit if you want to be unambiguous
- **`terracotta`** ‚Äî for **fourseasonsstudio**; terracotta brand + cream / brown neutrals
- **`orange`** ‚Äî for **garage-sales**; Tailwind orange-500 brand
- **`sand`** ‚Äî for **lessonforge**; warm sand neutrals with a blue accent
- **`navy`** ‚Äî for **melissapula.io** portfolio; navy brand
- **`emerald`** ‚Äî for **frula-homes** (fsbo-platform); emerald-green brand

In this Storybook, use the **Theme** dropdown in the toolbar at the top to flip any open story between themes. Or visit **Themes / Showcase** for a side-by-side comparison.

## Forms

Seven components participate in native HTML form submission via `ElementInternals`:

- `mfp-button` (type=submit/reset triggers form.requestSubmit / .reset)
- `mfp-input`, `mfp-textarea`, `mfp-select` (value submitted under `name`)
- `mfp-checkbox`, `mfp-switch`, `mfp-radio` (value submitted when checked/on)

Inside a `<form>`, `new FormData(form).get('email')` Just Works. `required` triggers native validation. `error` becomes a custom validity message.

```html
<form @submit.prevent="onSubmit">
    <mfp-input name="email" type="email" required label="Email" />
    <mfp-button type="submit" variant="primary">Sign up</mfp-button>
</form>
```

## A few architectural notes

- **Brand colors are themed via a semantic layer**, not literal hex. Component styles reach for `var(--color-brand-primary)`, not `#2563eb`. Themes only need to override ~5 brand tokens to re-skin the whole UI.
- **Status colors are universal across themes** ‚Äî `success` is green everywhere, `error` is red. These represent meaning, not brand.
- **Components default to safe fallback values** so they look reasonable even if you forget to load `tokens/css`. The fallbacks match the blue/default theme.
- **`tokens` is an _optional_ peer dependency** for every component. You can install one component package and ship without tokens; loading tokens just gives you the canonical look.
